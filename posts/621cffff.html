3、# 与 $ 区别

#{}

它表示一个占位符

  • 如果传入是简单类型，占位符的名称写value 或其它名称
  • 如果传入是复杂类型，占位符的名称写必须要与属性名一致
  • 动态表名不可以用 #{}：select * from #{table}
  • 动态列名不可以用 #{}：select #{column} from table
  • 动态排序列不可以用 #{}：select * from table order by #{column}

${}

它表示一个拼接符

  • 如果传的是简单类型，拼接符的名称只能写 value

    <delete id="deleteStudentById" >    
  delete from student where id = ${value}
</delete>

  • 如果传的是复杂类型，拼接符的名称必须要与属性名一致

  • ${} 占位符是字符串连接符，可以用来动态设置表名，列名，排序名

    1. 动态表名：select * from ${table}
    2. 动态列名：select ${column} from table
    3. 动态排序：select * from table order by ${column}

  • 不安全的，拼接会导致 Sql 注入，要慎用

    <select id="selectStudentsByName" resultType="Student">    	 select id,name,age,score 
  from student 
  where name like '%${value}%'
</select>

区别

#{}相当于jdbc中的preparedstatement(预编译)，${}是直接使用里面的值进行拼接

4、参数传递

4.1 默认参数传递

我们先看看接口是如何定义的，默认的传递方式我们只需要定义传统的方法就行。

public List<Student> findOrder( 
		String sort, 	//排序字段  
    String order,  	//排序方向  
    int start,      //起始记录  
    int limit     	//记录条数
 )

MyBatis默认会将第一个参数定义为param1后面依次类推param2、param3、param4。

<!-- 默认参数传递 -->
<select id="findOrder"  resultType="Student">
  select * from student order by ${param1} ${param2} limit #{param3} ,#{param4}
</select>

测试

@Test
public void testFindOrder(){
    SqlSession session = MyBatisUtils.getSession();
    StudentMapper mapper = session.getMapper(StudentMapper.class);
    List<Student> students =  mapper.findOrder("id","desc",0,10);
    System.out.println(students.get(0));
    MyBatisUtils.closeSession();
}

我们可以看到，我们没有写parameterType属性，默认MyBatis中的定义的param1、param2、param3、param4填写到对应Sql语句中。

4.2 注解方式参数传递

如果位置想随意放，可使用注解。我们看接口定义的方式：

public List<Student> findOrder2(
  @Param("order") String order,
  @Param("sort") String sort,
  @Param("start") int start,
  @Param("end")int end
);

即在每个参数前面加一个 @Param(“name”)，这样定义有什么好处，我们看MyBatis配置文件：

<select id= "find2"  resultType= "Student" >
 	select * from student 
  order by ${sort} ${order} 
  limit #{start}, #{end} 
</select>

测试

@Test
public void testFindOrder2(){
    SqlSession session = MyBatisUtils.getSession();
    StudentMapper mapper = session.getMapper(StudentMapper.class);
    List<Student> students =  mapper.findOrder("id","desc",0,10);
    System.out.println(students.get(0));

}

4.3 Map或对象方式传递

Map或对象的传递方式，实质是键值对方式传入，类对象我们也可以看做键值对，里面的属性可以看做键，属性对应的值可以看做值。这样我们在Sql中只要写明参数，MyBatis会帮你去找传递参数中的值，map方式是很简单，对象方式通过反射即可获取属性中的值，具体我们看下面的实例。

接口：

public  List<Student> findOrder3( Map<String,Object> data );

MyBatis配置文件：

<select id= "find2"  resultType= "Student" >
 	select * from student 
  order by ${sort} ${order} 
  limit #{start}, #{end} 
</select>

测试方法

@Test
public void testFindOrder3() {
    SqlSession session = MyBatisUtils.getSession();
    StudentMapper mapper = session.getMapper(StudentMapper.class);
    Map<String, Object> params = new LinkedHashMap<String,Object>();
    params.put("sort", "id");
    params.put("order", "desc");
    params.put("start", 0);
    params.put("end", 10);
    List<Student> student = mapper.findOrder3(params);
    System.out.println(student.size());
    MyBatisUtils.closeSession();
}

5、映射文件常用标签

MyBatis真正强大的地方在于它的映射语句。这是奇迹发生的地方，对于MyBatis所有的优点，MyBatis的xml配置文件是相当简单的。如果你拿它与jdbc相比，你将会节省95%的代码。MyBatis的构建就是聚焦于Sql的，使其远离于普通的方式。

Sql映射文件有很少的几个顶级元素（按照它们应该被定义的顺序）：

  • cache - 配置给定命名空间的缓存。
  • cache-ref – 从其他命名空间引用缓存配置。
  • resultMap – 最复杂，也是最有力量的元素，用来描述如何从数据库结果集中来加载你的对象。
  • parameterMap – 已经被废弃了！老式风格的参数映射。内联参数是首选，这个元素可能在将来被移除。这里不会记录。
  • Sql – 可以重用的Sql块，也可以被其他语句引用。
  • insert – 映射插入语句
  • update – 映射更新语句
  • delete – 映射删除语句
  • select – 映射查询语句

下面我们来看看各个元素的用法

5.1 resultMap

  • 自定义映射数据库 column 与 po 类中的 property 的关系
<!-- 实体到数据库之间的映射-->
<resultMap type= "Student"  id= "Student" >
    <id property= "id"  column= "id" />
    <result property= "age"  column= "age" />
    <result property= "name"  column= "name" />
</resultMap>

5.2 insert

  • 做添加功能，编写添加语句
  • 添加功能时，要返回自动增长列的值
    • 在映射文件中insert标签中加上两个属性
      • useGeneratedKeys=”true” 使用合成的键
      • keyProperty=”id” 指生成的键自属性列
<!-- 插入语句 -->
<insert id="addStudent" parameterType="Student" useGeneratedKeys="true" keyProperty="id">
	insert into student 
  values(null,#{name},#{age},#{gender},#{address})
</insert>

在mysql中如何插入数据后直接将插入的id带回来呢，方法很简单，只需要在insert的属性中添加useGeneratedKeys=”true” keyProperty=”id” ，其中keyProperty是实体对象中主键。当然对于其它数据会有所不同，大家可以看看MyBatis的官方文档，里面有详细的描述。

5.3 select

做查询功能，编写查询语句

<!-- 选择语句 -->

5.6 sql标签和include标签

定义一个sql片断，其它地方可用include标签包含它

<sql id="studentField">
	id,name,age
</sql>

<select id="selectByField" resultType="student" parameterType="int">
	select <include refid="studentField"/> 
  from student 
  where id=#{id}
</select>

上面我们定义了一个 Sql 语句，如果其它地方要用到与我们定义的Sql中相同内容的地方，我们只需要用<include refid="studentField”/>引入我们已经定义的 Sql 即可。

5.7 cache标签

<cache/> 配置二级缓存需要的标签

在默认情况下面，MyBatis是没有开启缓存，如果需要开启缓存我们需要在Sql配置文件中开启缓存，即：<cache/>。

那么开启缓存到底有什么好处呢？

  1. 映射语句文件中的所有 select 语句将会被缓存。
  2. 映射语句文件中的所有 insert，update 和 delete 语句会刷新缓存。
  3. 缓存会使用Least Recently Used（LRU，最近最少使用的）算法来收回
  4. 根据时间表（比如no Flush Interval，没有刷新间隔），缓存不会以任何时间顺序来刷新。
  5. 缓存会存储列表集合或对象（无论查询方法返回什么）的1024个引用。
  6. 缓存会被视为是read/write（可读/可写）的缓存，意味着对象检索不是共享的，而且可以安全地被调用者修改，而不干扰其他调用者或线程所做的潜在修改。
  7. 也可以自定义缓存，具体请参考官方文档。

8、返回自动增长或uuid

SELECT LAST_INSERT_ID()：得到刚insert进去记录的主键值，只适用与自增主键。

keyProperty：将查询到主键值设置到parameterType指定的对象的哪个属性

order：SELECT LAST_INSERT_ID()执行顺序，相对于insert语句来说它的执行顺序

resultType：指定SELECT LAST_INSERT_ID()的结果类型

  • MyBatis接口
public void insertStudent(Student student);
public void addStudent(Student student);
  • MyBatis映射

9.2 resultMap

如果查询出来的列名和 pojo 的属性名不一致，通过定义一个 resultMap 对列名和 pojo 属性名之间作一个映射关系。

  1. 定义 resultMap
  2. 使用 resultMap 作为 statement 的输出映射类型
<!--定义 resultMap，将 select id stu_id,name stu_name 和Student 类中的属性作一个映射关系
    type：resultMap最终映射的Java对象类型,可以使用别名
    id：对resultMap的唯一标识
-->
<resultMap id="StudentResultMap" type="net.wanho.po.Student" >
<!-- 
    id表示查询结果集中唯一标识 
    column：查询出来的列名
    property：type指定的pojo类型中的属性名
    最终resultMap对column和property作一个映射关系 （对应关系）
-->
    <id column="stu_id" property="id"/>
<!-- 
    result：对普通名映射定义
    column：查询出来的列名
    property：type指定的pojo类型中的属性名
    最终resultMap对column和property作一个映射关系 （对应关系）
-->
    <result column="stu_name" property="name"/>
</resultMap>
  • 使用resultMap作为statement的输出映射类型
<!--使用resultMap进行输出映射
    resultMap：指定定义的resultMap的id，如果这个resultMap在其它的mapper文件，前边需要加namespace
-->
<select id="findUserById" parameterType="int" resultMap="StudentResultMap">
  select id stu_id,name 
  from stu_name 
  WHERE id=#{id}
</select>

9.3 小结

使用resultType进行输出映射，只有查询出来的列名和pojo中的属性名一致，该列才可以映射成功。

如果查询出来的列名和pojo的属性名不一致，通过定义一个resultMap对列名和pojo属性名之间作一个映射关系。

10、对比输入映射和输出映射

  • 入参：

    • paramaterType
      • 值类型和string，有内存类型别名对应
      • 是其它引用类型，写类型的全限定名
    • 数组或集合，就不需要写入参类型了
      • 要遍历数组中的元素，需要使用foreach标签，传入collection值固定为array
      • 要遍历集合中的元素，需要使用foreach标签，传入collection值固定为list

  • 出参

    • resultType
      • 数据库的列别名与po的属性名一致，直接返回resultType
    • resultMap
      • 数据库的列别名与po的属性名不一致，需要自己编写resultMap的映射关系
      • 表链接也必须使用resultMap来构建关系

  • 高级映射：

    将关联查询的列映射到一个pojo属性中。（一对一）

    将关联查询的列映射到一个List中。（一对多）
