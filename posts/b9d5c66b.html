三、Mybatis 优化

  • 连接数据库的配置单独放在一个 properties 文件中 jdbc.properties
  • 为实体类定义别名,简化 Sql 映射 xml 文件中的引用

  • 打印 Sql 语句

1、连接池的配置

配置文件编码说明

  • 我们在连接mysql数据库的时候一般都会在url后面添加useUnicode=true&characterEncoding=UTF-8 ，但是问什么要添加呢？

  • 添加的作用是：指定字符的编码、解码格式。

    例如：mysql数据库用的是gbk编码，而项目数据库用的是utf-8编码。这时候如果添加了useUnicode=true&characterEncoding=UTF-8 ，那么作用有如下两个方面：

    • 存数据时：数据库在存放项目数据的时候会先用 UTF-8 格式将数据解码成字节码，然后再将解码后的字节码重新使用 GBK 编码存放到数据库中。
    • 取数据时：在从数据库中取数据的时候，数据库会先将数据库中的数据按 GBK 格式解码成字节码，然后再将解码后的字节码重新按 UTF-8格式编码数据，最后再将数据返回给客户端。
  1. 注意：在 xml 配置文件中配置数据库 url 时，要使用 & 的转义字符也就是&amp;
<property name="url" value="jdbc:mysql://localhost:3306/mydb?useUnicode=true&amp;characterEncoding=UTF-8" />

1.5 总结

  • 优先级
    • 数据源本身那
    • Properties对象
    • 外部文件 db.properties
    • 配置文件中Properties标签元素
  • 建议：
    • 不要在properties元素体内添加任何属性值，只将属性值定义在properties文件中。
    • 在properties文件中定义属性名要有一定的特殊性，如：jdbc.driver

4、优化SqlSessionFactoryBuilder、SqlSessionFactory和SqlSession

  • SqlSessionFactoryBuilder
    • 用于创建SqlSessionFactory的构建类，
    • 没用引用指向， SqlSessionFactoryBuilder会被gc回收。
    • 每次都new SqlSessionFactoryBuilder()，其实每次都创建了新的对象，需要开销
  • SqlSessionFactory
    • 每次都创建了新SqlSessionFactory对象，需要很大的开销，所以保留在整个应用的生命周期中
  • SqlSession
    • SqlSession对象就线程不安全的所以不应该被共享
    • 每一个线程都应该有自己的SqlSession对象
    • 执行增删、删除 和修改的时候手动commit
    • 常用方法
      • selectOne(返回单个对象)、
      • selectList（返回单个或多个对象）
      • update()
      • getMapper()
  • 如何优化
    • SqlSessionFactoryBuilder\SqlSessionFactory 全局
    • SqlSession 局部
  • uitl/MyBatisUtils.Java
public class MyBatisUtils {

    private static SqlSessionFactory sessionFactory;
    private static ThreadLocal<SqlSession> threadLocal;

    //静态代码块
    static {
        // 初始化本地线程对象
        threadLocal = new ThreadLocal<SqlSession>();

        // 创建了全局的SqlSessionFactoryBuilder和SqlSessionFactory
        try {
            InputStream inputStream = Resources.getResourceAsStream("MyBatis-config.xml");
            sessionFactory = new SqlSessionFactoryBuilder().build(inputStream);
        } catch (IOException e) {
            throw new DaoException(e);
        }
    }

    /**
     * 获取SqlSession
     * @return
     */
    public static SqlSession getSession(){
        // 先从本地线程中获取
        SqlSession SqlSession = threadLocal.get();
        // 获取不到，再从SqlSessionFactory中得到
        if(SqlSession==null){
            // 参数false也是默认值，代表要手动提交
            SqlSession = sessionFactory.openSession(false);
            // 获取到对象之后，加入ThreadLocal
            threadLocal.set(SqlSession);
        }
        return  SqlSession;
    }

    /**
     * 释放SqlSession
     */
    public  static  void closeSession(){
        // 先从本地线程获取连接
        SqlSession SqlSession = threadLocal.get();
        if(SqlSession!=null) {
            // 关闭联系，且从本地线程中移除
            SqlSession.close();
            threadLocal.remove();
        }
    }
}

4.1 本地线程回顾

  • 示例一

    package net.wanho.threadlocal;

public class Demo1 {

	public static void main(String[] args) {
		MyRunnable r = new MyRunnable(new TestNum());
		new Thread(r).start(); //1 2 3
		new Thread(r).start(); //4 5 6
		new Thread(r).start(); //7 8 9
	}

}

class TestNum{
	private int num = initNum(); 

	private int initNum() {
		System.out.println("inital TestNum...");
		return 0;
	}

	public int getNum() {
		return ++num; 
	}
}

class MyRunnable implements Runnable{
	private TestNum testNum;

	public MyRunnable(TestNum testNum) {
		this.testNum = testNum;
	}


	@Override
	public void run() {
		for(int i=0;i<3;i++) {
			System.out.print(testNum.getNum()+"\t"); //1 2 3
		}
	}

}

  • 示例二

    package net.wanho.threadlocal;

public class Demo2 {
	public static void main(String[] args) {
		MyRunnable2 r = new MyRunnable2(new TestNum2());
		new Thread(r).start(); //1 2 3
		new Thread(r).start(); //4 5 6
		new Thread(r).start(); //7 8 9
	}
}

class TestNum2{
	private ThreadLocal<Integer> num = new ThreadLocal<Integer>() {
		@Override
		protected Integer initialValue() {
			System.out.println("inital TestNum...");
			return 0;
		}
	};
	public int getNum() {
		num.set(num.get()+1);
		return num.get();
	}
}

class MyRunnable2 implements Runnable{
	private TestNum2 testNum2;
	public MyRunnable2(TestNum2 testNum2) {
		super();
		this.testNum2 = testNum2;
	}
	@Override
	public void run() {
		for(int i=0;i<3;i++) {
			System.out.print(testNum2.getNum()+"\t");
		}
	}
}
